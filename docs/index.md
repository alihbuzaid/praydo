## Project Overview

Praydo is a cross-platform desktop prayer times application. The frontend computes schedules locally and renders the full user experience, while the Tauri shell provides native packaging, tray behavior, permissions, notifications, and bundled resources.

## Project Structure

This project consists of 2 parts:

### Frontend (`frontend`)

- **Type:** web
- **Location:** `src`
- **Tech Stack:** SvelteKit 2, Svelte 5, TypeScript, Skeleton UI, Tailwind CSS 4, Vitest
- **Entry Point:** `src/routes/+layout.svelte`

### Desktop Shell (`desktop-shell`)

- **Type:** desktop
- **Location:** `src-tauri`
- **Tech Stack:** Tauri 2, Rust 2021, Cargo, Tauri plugins
- **Entry Point:** `src-tauri/src/main.rs`

## Cross-Part Integration

The frontend runs inside Tauri and communicates with the shell over the Tauri bridge. The main custom bridge contract is the `send_native_notification` command, while the rest of the integration uses Tauri plugins for HTTP, notifications, autostart, and resource filesystem access.

## Getting Started

### Frontend Setup

**Prerequisites:** Node.js, `pnpm`

**Install & Run:**

```bash

pnpm dev
```

### Desktop Shell Setup

**Prerequisites:** Node.js, `pnpm`, Rust toolchain, Tauri OS prerequisites

**Install & Run:**

```bash
pnpm install
pnpm tauri dev
```
